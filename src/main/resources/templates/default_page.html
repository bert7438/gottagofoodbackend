Gotta Go Food Backend

Database control panel

Можно отправлять запросы по этому адресу, чтобы добавлять, удалять, изменять и смотреть записи в базе данных!

User

/user/add?name="name" - добавляет в БД пользователя с именем "name"

/user/del?id="id" - удаляет из БД пользователя с идентификатором(не токеном!) "id"

/user/get?id="id" - получает JSON-строковое представление записи с идентификатором(не токеном!) "id"

/user/getall - получает ВСЕ записи БД в JSON-строковом представлении

Product

/product/add?name="name&size="size&ev="ev"&prot="prot"&f="f"&carb="carb&price="price" - добавляет в БД продукт с именем "name", размером(объемом или массой) "size", энергетической ценностью "ev" в 100 г, белками "prot" в 100 г, жирами "f" в 100 г, углеводами "prot" в 100 г, стоимостью "price" рублей. ВСЕ ПАРАМЕТРЫ ДОЛЖНЫ БЫТЬ УКАЗАНЫ, ИНАЧЕ ПРОДУКТ НЕ БУДЕТ ДОБАВЛЕН В БД! Параметры энергетической ценности, белков, жиров, углеводов и цены могут быть дробными

/product/del?id="id" - удаляет из БД продукт с идентификатором "id"

/product/get?id="id" - получает JSON-строковое представление записи с идентификатором "id"

/product/getall - получает ВСЕ записи БД в JSON-строковом представлении

LIST GENERATION

/gen/listdev - возвращает json представление списка покупок. Генеруются 3 покупки из базы данных:основное блюдо, дополнительное блюдо и напиток. Пока генерируется простым рандомом.
